# Turbine-Based Ventilator Prototype Firmware (2019)

This repository contains the firmware for a turbine-based medical ventilator prototype developed in 2019. It runs on an 8-bit AVR microcontroller (such as the Arduino Mega 2560) and demonstrates advanced fluid dynamics compensation, high-frequency motor control, and complex state-machine logic for respiratory support.

> **⚠️ MEDICAL DISCLAIMER:** 
> This code is published for educational, historical, and open-source research purposes only. It is a prototype and has **not** been subjected to regulatory approval (e.g., FDA, CE, ISO). **Do not use this code for human clinical use, life-support applications, or direct patient care.**

## 🌟 Key Features

* **Advanced Breathing Modes:** 
  * Continuous Positive Airway Pressure (**CPAP**)
  * Pressure Control (**PC-CMV**, **PC-AC**)
  * Volume Control (**VC-CMV**, **VC-AC**)
* **Patient Triggering (Assist-Control):** Implements a debounced pressure-drop trigger (-0.2 cmH2O) to detect patient inhalation efforts without being susceptible to sensor noise.
* **Non-Linear PID Control:** Utilizes a custom velocity-form PID controller with dynamic gains to overcome the mechanical inertia of the turbine blower, ensuring rapid spin-up and stable pressure maintenance.
* **Dynamic Flow Calibration:** Compensates for non-linear flow dynamics by referencing precision calibration arrays (for both Air and Oxygen) stored efficiently in `PROGMEM` to save vital SRAM.
* **Temperature Compensation:** Integrates live temperature readings with binary-search lookup tables to correct flow calculations dynamically.
* **High-Frequency Control Loop:** Runs a strict 100 Hz (10ms) control loop, managing hardware Timer 4 for high-resolution PWM motor control.

## 🛠 Hardware Architecture

Based on the pinout and peripheral usage in the code, this firmware is designed for the following hardware ecosystem:

* **Microcontroller:** Arduino Mega 2560 (Requires Timer 4 for hardware PWM and multiple analog inputs).
* **Turbine / Blower:** Driven via high-frequency PWM generated by Timer 4 (OCR4C / Pin 8).
* **Pressure Sensor:** I2C digital differential pressure sensor (Address `0x78`).
* **Air Flow Sensor:** Analog flow sensor (Pin `A3`) with thermistor/temperature output (Pin `A4`).
* **Oxygen Flow Sensor:** Analog flow sensor (Pin `A7`) with thermistor/temperature output (Pin `A8`).

## 📁 Firmware Structure

* `setup()`: Initializes I2C, Serial, extracts floating-point calibration data from Big-Endian PROGMEM arrays, sets up Timer 4 PWM, and calculates initial mode parameters.
* `loop()`: Acts as the main router, delegating to the active mode's state machine (CPAP, VC, or PC).
* `calculate_required_parameter()`: Translates clinical targets (Target RR, I:E ratio, Tidal Volume, PEEP) into machine-level timings (Inspiratory time, Expiratory time, Target Flow Rate).
* **State Machines:** Each breathing mode (e.g., `volume_control_mode_process`) is governed by an explicit state machine managing Inspiratory, Hold, Blow-off, and Expiratory phases.
